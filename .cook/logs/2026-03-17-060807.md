## Review: Earthbound Palette Restyle (Iteration 1/3)

The changes are a comprehensive, consistent palette swap from a neon/CRT aesthetic to a warm Earthbound pixel-art style, touching both `style.css` and the game canvas in `index.html`.

---

### High Severity

**None found.** The palette swap is complete and internally consistent. No broken references, no missing variable definitions, no rendering errors.

---

### Medium Severity

1. **Stale class naming** (`style.css:56-58`, `index.html` throughout) — Classes like `.neon-cyan`, `.neon-magenta`, `.btn-cyan`, `.btn-magenta` no longer correspond to their visual meaning. "Cyan" now renders as steel blue, "magenta" as warm red. The `--cyan`/`--magenta` variable aliases (line 20-21) keep them functional, but the naming is misleading for anyone reading the markup. Not a runtime bug, but a maintenance concern.

2. **`body::after { display: none; }` instead of removal** (`style.css:53`) — The CRT scanlines pseudo-element is hidden but still generated. Since this file fully replaces the old CSS, the `body::after` rule could simply be deleted rather than hidden. The browser still creates the element in the DOM, though the performance impact is negligible.

---

### Low Severity

1. **Hardcoded colors outside the variable system** — `hero` background gradient uses raw hex `#181830`, `#20204a`, `#282860` (line 87), and `pre` background uses `#0c0c1c` (line 256). These could reference `var(--bg)` / `var(--bg-alt)` for easier future theming, though the current values are intentional near-variants.

2. **Redundant `::before` rules on `.link-card`** (`style.css:169-171`) — Both `.link-card::before { display: none; }` and `.link-card:hover::before { display: none; }` exist. The non-hover rule alone is sufficient since it applies in all states.

3. **Dead code comment** (`style.css:52`) — `/* remove CRT scanlines */` is a transitional comment that won't make sense to a future reader who never saw the CRT version. Minor documentation noise.

---

### Summary

The restyle is well-executed with no functional issues. The Earthbound palette is applied consistently across CSS tokens, utility classes, card/button/nav components, canvas rendering, and the game HUD. All `text-shadow` glows and `box-shadow` blurs have been cleanly replaced with flat colors and hard pixel shadows. The medium-severity items are naming/cleanup concerns, not bugs.
